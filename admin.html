🛠 Кабинет администратора

Кабинет администратора управляет структурой всей школы: пользователями, предметами, классами, расписанием, магазином, AI-контролем, общими рассылками и режимом каникул. Это самый чувствительный кабинет, поэтому доступ к нему должен быть только у владельца ADMIN_ID.

Главный экран администратора

Безопасность администратора

  • В админский кабинет должен попадать только пользователь с заранее заданным ADMIN_ID.
  • Даже если кто-то узнает регистрационный код роли администратора, сам кабинет не должен открываться постороннему пользователю.
  • Сервисная команда сброса должна оставаться скрытой и использовать секретный суффикс, например /reset_секрет.

👥 Пользователи

  1. Администратор открывает раздел управления пользователями.
  2. Выбирает, кого хочет перевести обратно в гостя: учителя, родителя или ученика.
  3. Для учеников сначала выбирается класс, затем конкретный ребенок.
  4. После подтверждения роль пользователя сбрасывается до гостя.
Это не удаление Telegram-аккаунта, а сброс роли внутри бота. Пользователь сможет снова зарегистрироваться по нужной роли.

🧠 AI-Контроль

  1. Раздел Пульс школы показывает последние AI-запросы учеников в реальном времени с пагинацией.
  2. Раздел Поиск по ученику позволяет выбрать класс, затем ученика и посмотреть его AI-историю.
  3. Раздел очистки полностью удаляет историю AI-запросов после отдельного подтверждения.
Очистка AI-истории необратима. Эту кнопку нужно описывать в инструкции отдельно и очень явно.

🏪 Товары магазина

  1. Администратор открывает магазин.
  2. Для добавления товара вводит название, цену и фотографию.
  3. Если фото не нужно, вместо него можно отправить -.
  4. Для удаления выбирает товар из списка.

🏫 Добавить класс, 📚 Добавить предмет, 🔗 Привязать предмет

Это базовый блок настройки школы. Сначала создаются классы и предметы, затем предметы привязываются к конкретным классам. Пока привязки нет, учитель и ученик не увидят этот предмет в своих сценариях.

📅 Расписание классов

Раздел показывает уже готовое недельное расписание выбранного класса. Это режим просмотра, без редактирования.

🗓 Настроить расписание

  1. Администратор выбирает класс.
  2. Выбирает день недели.
  3. Далее работает в одном из двух режимов.
Ручной режим
Выбрать номер урока, затем выбрать предмет и сохранить слот.
Быстрый режим
Находясь в состоянии выбора урока, отправить сообщением список предметов через запятую. Бот постарается сам сопоставить их с базой и создать расписание блоком.
Настройка расписания администратором

📢 Рассылка

  1. Администратор нажимает 📢 Рассылка.
  2. Выбирает аудиторию: учителя, родители, ученики или все сразу.
  3. Отправляет текст, фото, документ или иной поддерживаемый контент.
  4. Бот рассылает сообщение и показывает итоговую статистику доставки.

🛑 Режим каникул

  1. Администратор открывает раздел каникул.
  2. Если каникулы не включены, бот показывает обычный рабочий режим.
  3. Для включения вводится дата окончания в формате ДД.ММ.
  4. Для выключения используется отдельная кнопка остановки режима.
Режим каникул особенно важен для напоминаний и школьной коммуникации. Его полезно описать в инструкции как организационный, а не декоративный режим.

🎯 Цели класса

  1. Администратор выбирает класс.
  2. Вводит название цели, например поездка, техника, экскурсия или подарок.
  3. Вводит требуемую сумму.
  4. Добавляет фото или отправляет -, если фото не нужно.
  5. После этого ученики видят цель в разделе 💰 Копилка.
В один момент времени у класса должна быть только одна активная цель. Именно на эту механику завязаны ученические пожертвования монет.

Частые ошибки администратора

  • Учитель не видит предмет. Предмет создан, но не привязан к классу.
  • Ученик не видит расписание. Либо расписание не настроено, либо ученик не привязан к классу.
  • Рассылка доставилась не всем. Некоторые пользователи не открывали чат с ботом или заблокировали его.
  • В AI-контроле пусто. Значит, запросов пока не было либо история уже очищена.
