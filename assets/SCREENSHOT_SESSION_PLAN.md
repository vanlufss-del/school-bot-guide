# План сессии съемки

Это маршрут съемки для инструкции.

Используйте его вместе с:
- `docs/assets/SCREENSHOT_CHECKLIST.md`
- `docs/guide.html`

Главный принцип:
- не прыгать хаотично между экранами
- идти по ролям последовательно
- после каждого кадра сразу сохранять файл под финальным именем

## Перед началом

Подготовьте четыре тестовых аккаунта:
1. один ученик
2. один учитель
3. один родитель, привязанный к ученику
4. один администратор с реальным доступом в админку

Подготовьте данные заранее:
- существует хотя бы один класс
- в одном классе есть минимум 2 ученика для PvP и рейтинга
- есть хотя бы один учитель
- есть хотя бы один привязанный родитель
- предметы уже привязаны к классу
- расписание настроено хотя бы на несколько дней недели
- есть хотя бы одно домашнее задание
- есть хотя бы несколько оценок
- есть хотя бы одна цель класса для копилки
- есть хотя бы одна родительская цель, если хотите отдельно снять экран целей ученика
- есть хотя бы один товар магазина
- есть хотя бы один AI-запрос, чтобы AI-контроль админа не был пустым

Рекомендуемый порядок работы:
1. Подготовить данные через админа и учителя.
2. Снять все экраны ученика.
3. Снять все экраны учителя.
4. Снять все экраны родителя.
5. Снять все экраны администратора.
6. Открыть `guide.html` и проверить или вставить изображения.

## Сессия 1: администратор готовит данные

Зайдите под администратором и убедитесь, что система готова к съемке.

### Блок A. Базовая структура
1. Откройте `🔧 Админ-панель`.
2. Если нужно, создайте класс через `🏫 Добавить класс`.
3. Если нужно, создайте предметы через `📚 Добавить предмет`.
4. Привяжите предметы к классу через `🔗 Привязать предмет`.
5. Настройте недельное расписание через `🗓 Настроить расписание`.
6. Откройте `📅 Расписание классов` и проверьте, что расписание видно.

### Блок B. Мотивационные данные
1. Откройте `🎯 Цели класса`.
2. Создайте одну активную цель класса с названием, суммой и, при желании, фото.
3. Откройте `🏪 Товары магазина`.
4. Добавьте хотя бы один тестовый товар, чтобы экраны магазина не были пустыми.

### Блок C. Коммуникация и AI-данные
1. Один раз откройте `📢 Рассылка`, чтобы позже можно было снять экран выбора аудитории.
2. Откройте `🧠 AI-Контроль` и убедитесь, что меню доступно.
3. Если AI-история пустая, попросите ученика задать 2-3 вопроса AI-репетитору до начала съемки админских экранов.

## Сессия 2: учитель готовит учебный контент

Зайдите под учителем.

### Блок A. Домашнее задание
1. Откройте `🏫 Учебный процесс`.
2. Откройте `📝 Добавить задание`.
3. Выберите класс.
4. Выберите предмет.
5. Создайте одно домашнее задание с понятным текстом.
6. При необходимости создайте еще одно задание с фото, чтобы в инструкции был и текстовый, и медиа-сценарий.

### Блок B. Оценки
1. Откройте `⭐️ Журнал`.
2. Выберите класс и предмет.
3. Выберите сегодня или вчера.
4. Поставьте несколько оценок хотя бы одному ученику.
5. Сохраните хотя бы одну оценку с комментарием.
6. Сохраните хотя бы одну оценку без комментария.

### Блок C. Отчеты и проверка
1. Откройте `🔍 Проверка ДЗ` и убедитесь, что хотя бы один ученик отметил домашнее задание как выполненное.
2. Откройте `📉 Отчеты` и убедитесь, что экран выбора месяца работает.
3. Если возможно, подготовьте достаточно данных, чтобы Excel-отчет реально формировался.

## После съемки

1. Откройте `docs/assets/` и убедитесь, что имена файлов совпадают с чек-листом.
2. Удалите случайные дубликаты вроде `IMG_1234.png` или `final-final-screen.png`.
3. Если какой-то кадр размыт, переснимите его сразу.
4. Откройте `docs/guide.html` и сверьте имена изображений с реальными файлами.
5. Только после этого меняйте ссылки инструкции в самом боте.

## Важные ограничения

Не делайте этого во время реальной рабочей сессии школы:
- не очищайте AI-историю ради скриншота
- не удаляйте реальных пользователей ради скриншота
- не включайте каникулы без согласования
- не запускайте реальную массовую рассылку без предупреждения аудитории

Если нужно, используйте отдельную демонстрационную базу или контролируемое тестовое окружение.
