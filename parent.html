👶 Добавить ребенка

  1. Родитель нажимает 👶 Добавить ребенка.
  2. Бот просит ввести ID ребенка.
  3. Родитель вводит числовой ID, который ученик видит в своем профиле.
  4. При успешной привязке бот подтверждает, что ребенок добавлен.
Если введен нечисловой ID или такого ученика нет в системе, бот покажет ошибку. В инструкции важно отдельно показать, где ученик берет свой ID.

📊 Успеваемость

  1. Родитель нажимает 📊 Успеваемость.
  2. Выбирает одного из привязанных детей.
  3. Бот формирует детальную карточку по всем предметам.
  4. Внутри карточки отображаются оценки, средний балл, статус домашнего задания, XP, монеты и текущее звание.

Это основной управленческий раздел родителя. После выбора ребенка открывается специальное меню действий.

🤖 AI-Анализ недели

  1. Родитель выбирает ребенка.
  2. Нажимает 🤖 AI-Анализ недели.
  3. Бот собирает данные за неделю: оценки, пропуски, активность в AI, XP и монеты.
  4. После короткой паузы бот выдает структурированный педагогический отчет в уважительном тоне.
Этот отчет полезен не только как сводка, но и как шаблон разговора с ребенком: в нем уже есть сильные стороны, риски и конкретный педагогический совет.

🎯 Поставить цель

  1. Родитель выбирает ребенка.
  2. Нажимает 🎯 Поставить цель.
  3. Выбирает предмет.
  4. Выбирает целевую оценку: 8, 9 или 10.
  5. Пишет награду в свободной форме: деньги, поездка, подарок, поход в кафе и так далее.
  6. Бот сохраняет цель и отправляет уведомление ученику.
По одному предмету одновременно может быть только одна активная цель. Если цель уже есть, бот предупредит об этом и не создаст дубликат.

📜 Активные цели

Раздел показывает уже поставленные цели: предмет, требуемую оценку, награду и имя родителя, который создал цель. Это нужно, чтобы не дублировать обещания и видеть текущую мотивацию ребенка.

🤒 Заболел / Пропуск

  1. Родитель открывает карточку ребенка.
  2. Нажимает 🤒 Заболел / Пропуск.
  3. Выбирает срок: 🤒 1 день, 🤒 3 дня, 🤒 Неделя.
  4. Если ребенок вернулся, используется кнопка ✅ Выздоровел (Включить).
Смысл режима в том, чтобы временно не беспокоить уведомлениями. Это не удаляет ребенка и не меняет его оценки, а только влияет на коммуникацию.

❌ Отвязать

Отвязка удаляет связь между родителем и выбранным учеником. Если позже нужно будет снова получать данные, ребенка придется привязать заново по его ID.

Частые ошибки родителя

  • Не удается добавить ребенка. Чаще всего введен неверный ID или ребенок еще не зарегистрирован в системе.
  • Нет данных по успеваемости. У ученика может не быть привязанного класса, предметов или оценок.
  • AI-анализ слишком долго формируется. Это связано с генерацией текста AI; обычно нужно подождать несколько секунд.
  • Цель не ставится. Вероятнее всего, по этому предмету уже есть активная цель.
